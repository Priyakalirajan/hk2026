Hackathon Scope

What's Live vs. Simulated

Honest breakdown of what works end-to-end and what runs on mock/demo data

5-Step Onboarding Form UI
Fully functional guided form with auto-save, inline validation, PAN/GST regex checks, and resumable sessions.
KYC API Integration Layer
Integration points built. Mock mode active (MOCK_MODE=true in kycService.js) — realistic latency & responses simulated.
5-Dept Approval Workflow Engine
Full routing logic built: Sales → Finance → Legal → Credit → IT. Approve / Reject / RFI actions functional.
Admin Dashboard + OCR Panel
Side-by-side document/OCR viewer, TAT monitor, KPI pipeline view, and role-based access control all functional.
Real-Time Application Tracker
8-stage timeline with timestamps, completion %, estimated TAT, and PDF status export functional.
Notification Engine (Simulated)
SMS, Email & Push trigger logic built. Sandbox credentials in .env — templates fire on every stage change in demo.
Live KYC APIs (Production Keys)
PAN, Aadhaar, GST & Penny Drop run on sandbox mock data. Production keys require RBI-compliant AUA agreement — post-hackathon.
Live BC 365 ERP Push
ERP bridge code complete. Live push requires OBL's BC 365 company ID & Azure AD OAuth credentials — simulated in demo with mock dealer code generation.
Real Dealer / Transaction Data
Demo uses synthetic data from PNRao_SalesRetail_Customer_Master.xlsx & Dealership Form as test case templates. No live business records.
WhatsApp & Multi-Language (v1.1)
WhatsApp Business API integration and Hindi/regional language support are scoped for v1.1 — explicitly out of hackathon scope.
